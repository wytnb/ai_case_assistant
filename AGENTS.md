# AGENTS.md

本文件定义 Codex、Cursor 等 AI 编码代理在本仓库中的项目级执行规则。

## 1. 阅读顺序

开始规划或编辑前，按以下顺序阅读：

1. `README.md`
2. `AGENTS.md`
3. `docs/00-index.md`
4. `docs/docs-policy.md`
5. `docs/` 下与当前任务最相关的编号文档
6. 当前修改模块附近的代码与测试

如文档已存在，不要只凭代码自行假设需求含义。

## 2. 事实来源

- 业务、系统、流程、接口、测试与发布事实记录在 `docs/`。
- 本文件与 `.cursor/rules/` 只负责执行规则，不承载完整业务事实。
- 如果文档与代码冲突，不要静默忽略：
  - 先识别冲突点；
  - 若意图明确，在同一任务中同步更新文档和实现；
  - 若意图不明确，保持改动最小并在最终汇报中明确写出歧义。

## 3. 文档决策规则

### Rule A: 优先更新已有文档

默认先更新已有文档，不默认新建。

### Rule B: 以 `docs/docs-policy.md` 为准

当你判断“该改哪份文档”“是否需要新增文档”时，`docs/docs-policy.md` 是项目内的规范基线。
其中包含：

- 完整文档清单
- 模板
- 示例
- 命名与放置规则
- 新建文档判定逻辑

### Rule C: 新建文档后的必做项

如果新增文档，必须同步更新：

- `docs/00-index.md`
- `README.md` 中受影响的入口说明
- 相关规则文件中的引用

## 4. 强制文档同步

只要改动会影响以下任一方面，必须在同一任务中同步更新文档：

- 项目概览或范围
- 业务流程
- 领域对象或字段语义
- API 契约或 AI JSON
- 数据模型、迁移、本地文件路径
- 架构边界、模块职责、依赖方向
- 运行方式、环境变量、验证命令
- 测试策略、回归矩阵、发布 smoke

如果任务形成了新的重要架构决策，新增或更新 `docs/adr/*.md`。

如果任务改变了需求理解、范围边界、阶段性取舍或发布策略，更新 `docs/13-requirement-deltas.md`。

## 5. 强制测试与验证同步

### 新功能

至少补：

- happy path
- 一个边界场景
- 一个失败路径

### Bug 修复

补或更新一个回归测试，保证修复前会失败。

### 契约变更

补或更新相关接口 / 集成式验证。

### 真实接口 / 手工 smoke 触发条件

如果任务涉及以下任一情况，必须评估并在可行时执行真实接口验证或手工 smoke：

- AI 请求体或响应解析变化
- `features/ai/`、`core/network/`、`core/config/` 相关逻辑变化
- `AI_API_BASE_URL`、`RUN_REAL_AI_API_TESTS`、`USE_MOCK_AI_EXTRACT` 等环境变量变化
- 发布链路、演示链路、关键页面主入口变化
- 本地无法充分替代真实服务或真实设备的行为

如果没有执行真实接口验证或手工 smoke，最终汇报必须明确写出：

- 未执行的验证类型
- 未执行原因
- 剩余风险

## 6. 完成定义

除非用户明确只要方案，否则任务不能算完成，直到以下条件同时满足：

1. 代码或文档改动已完成；
2. 受影响文档已同步；
3. 需要的测试已新增或更新；
4. 本地自动化验证已执行，或阻塞原因已明确报告；
5. 需要时，真实接口验证或手工 smoke 已执行，或跳过原因已明确报告；
6. 最终汇报包含变更文件、测试、命令、验证类型和剩余风险。

不要在未验证时声称“已完成”。

## 7. 执行顺序

默认按以下顺序工作：

1. 先理解当前文档
2. 再核对代码与测试事实
3. 先收敛文档与计划
4. 再实施改动
5. 再补测试
6. 再执行本地验证
7. 需要时执行真实接口验证或手工 smoke
8. 最后汇报结果

如果需求仍模糊，优先补清文档，而不是直接做大范围代码实现。

## 8. 范围控制

避免无关重构。

如果你发现本任务之外的结构问题：

- 在最终汇报中说明；
- 只有在它阻塞当前任务或文档策略明确要求时才一起处理。

## 9. 输出要求

最终汇报至少包含：

- 修改了哪些代码文件
- 修改了哪些文档文件
- 是否新增文档，以及原因
- 新增或更新了哪些测试
- 本地自动化验证运行了哪些命令
- 手工 smoke 运行了哪些步骤
- 真实接口 / 线上验证运行了哪些命令或步骤
- 哪些验证未执行，以及原因
- 结果摘要
- 剩余风险 / 后续建议

## 10. 禁止事项

不要：

- 编造代码、配置、测试或上游接口事实
- 在没有长期价值时新增大段专题文档
- 改了行为却不更新文档
- 修复 bug 却跳过回归测试
- 涉及真实接口或手工 smoke 的高风险变更却不说明验证策略
- 在未验证时声称任务完成
- 引入 CRLF；所有文本文件必须保持 LF
