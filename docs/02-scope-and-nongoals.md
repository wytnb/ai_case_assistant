# 范围与非目标

## 当前范围

当前 MVP 纳入范围的能力如下：

- 首页提供记录列表、新增记录、报告页三个入口
- 用户可填写 `rawText` 并可选选择多张图片
- 新增记录时先调用 `/ai/extract`，发送 `rawText` 与单一 `eventTime`
- `rawText` 在创建链路中按 `trim()` 后校验为非空且不超过 1000 字
- 提取成功后将健康事件和附件写入本地数据库与应用私有目录
- 列表页可查看本地记录摘要、单一事件时间和来源
- 详情页可查看原始文本、摘要、备注和图片附件预览
- 报告页可切换 `week`、`month`、`quarter`
- 报告生成调用 `/ai/report`，结果写入本地 `reports` 表并在详情页展示

## 当前不做

当前阶段默认不做以下能力，除非用户单独提出：

- 云端图片存储或云端数据库
- 账号登录、多用户、多档案
- OCR、图片内容提取、语音输入
- 在线问诊、自动诊断、医疗机构对接
- 记录编辑 / 删除
- 设置页、权限体系、后台管理
- 自动后台生成报告

## 版本边界

### 当前 MVP

- Android 演示优先
- 单用户、本地优先
- AI 只负责辅助整理与汇总，不做诊断
- 页面围绕“记录 -> 浏览 -> 报告”主链路展开

### 后续版本方向

以下方向在产品讨论中存在，但当前仓库未实现：

- 更完整的首页概览
- AI 追问与更细粒度结构化结果
- 更完整的附件管理
- 记录编辑 / 删除
- 数据导出、备份、隐私保护增强

## 延期项 / 候选项

- AI 追问页与追问会话落库
- 独立提取结果表
- 就医前摘要
- 报告 Markdown 富渲染
- 报告 mock 通道

## 已知约束

### 技术约束

- 数据当前仅保存在本地 Drift 和应用私有目录
- AI 提取与报告生成依赖外部 HTTP 接口
- 当前 `HealthEvent.sourceType` 统一保存为 `text`
- 图片附件不会参与 `/ai/extract`
- 出站 `eventTime` 统一序列化为不带毫秒、带 `+08:00` 的 ISO 8601 字符串

### 运行约束

- 仓库有多平台骨架，但当前验证重点是 Android
- 没有 CI、没有正式发布流水线、没有监控平台
- Android / iOS / macOS / Web 的应用标识仍有默认占位配置

### 文档约束

- 历史需求变化没有完整回填记录
- 任何无法从仓库确认的历史原因必须标记为“待确认”
