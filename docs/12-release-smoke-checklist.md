# 发布 Smoke 清单

当前仓库没有正式 CI/CD 或线上发布流程。本清单用于本地演示版、手工安装包或阶段性发布候选的最小 smoke 检查。

## 发布前检查

- [ ] `README.md`、`AGENTS.md`、`docs/00-index.md` 已同步最新事实
- [ ] `fvm flutter analyze` 通过
- [ ] `fvm flutter test` 通过
- [ ] 若改动了 Drift 表或数据库逻辑，已重新生成代码
- [ ] 已确认当前演示使用真实 AI 还是 `USE_MOCK_AI_EXTRACT=true`
- [ ] 若使用真实 AI，已确认 `AI_API_BASE_URL` 可访问
- [ ] 若本次先用 mock 验证，mock 通过后已执行带 `RUN_REAL_AI_API_TESTS=true` 的真实 AI 集成测试
- [ ] 演示设备或模拟器可正常启动应用
- [ ] 若 Android 真机依赖 Clash 等代理访问 `https://ai-api-worker.wytai.workers.dev`，已保留代理并按该网络前提执行 smoke

## 发布后检查

- [ ] 应用可正常打开首页
- [ ] 首页可进入记录列表、新增记录、报告页
- [ ] 本地数据库可正常读写，不出现启动即崩溃
- [ ] 关键页面无明显白屏、空白容器或死循环加载
- [ ] 日志中无持续刷新的严重错误

## 核心闭环验证

- [ ] 在新增记录页输入仅空白字符，确认无法提交并提示“请输入原始描述”
- [ ] 在新增记录页输入 1001 字，确认无法提交并提示“原始描述不能超过1000字”
- [ ] 新增一条 1000 字以内的纯文本记录，成功返回列表并看到“记录已保存”
- [ ] 打开刚创建的记录详情，能看到单一 `事件时间`、原始文本、摘要和备注 / 暂无备注
- [ ] 若本次演示包含图片附件，详情页能看到图片预览或明确的加载失败降级
- [ ] 生成一份周报、月报或季报，并能在列表中看到该报告
- [ ] 打开报告详情，能看到标题、摘要、建议和 Markdown 正文
- [ ] 若本次演示使用真实 AI，可通过日志或抓包确认 `/ai/extract` 与 `/ai/report` 都发送单一 `eventTime`，且为不带毫秒的 `+08:00` 格式
- [ ] 关闭并重新打开应用后，已有记录和报告仍可读取
- [ ] 若真机在保留代理的前提下仍无法跑通，已追加 Web Chrome 备用 smoke，并在结果中注明其不覆盖 Android 安装、附件与设备特有行为

## 监控项

当前没有正式监控平台。发布 / 演示阶段只能做以下人工观察：

- 关键命令是否通过
- 应用日志是否出现连续网络或数据库异常
- UI 是否出现主链路级别的失败提示

## 回滚条件

- 首页或主入口无法打开
- 新增记录主链路连续失败
- 报告生成主链路连续失败
- 数据升级后旧记录不可读
- 本地数据库或附件读取导致应用崩溃

当前没有自动回滚系统；若触发以上条件，只能回退到上一版可运行构建或暂停当前演示版发出。
