# 发布 Smoke 清单

当前仓库没有正式 CI/CD 或线上发布流程。
本清单用于本地演示版、手工安装包或阶段性发布候选的最小 smoke 检查。

## 发布前检查

- [ ] `README.md`、`AGENTS.md`、`docs/00-index.md` 已同步最新事实
- [ ] `fvm flutter analyze` 通过
- [ ] `fvm flutter test` 通过
- [ ] 若改动了 Drift 表或数据库逻辑，已重新生成代码
- [ ] 已确认当前演示使用真实 AI 还是 `USE_MOCK_AI_EXTRACT=true`
- [ ] 若使用真实 AI，已确认 `AI_API_BASE_URL` 可访问
- [ ] 演示设备或模拟器可正常启动应用

## 发布后检查

- [ ] 应用可正常打开首页
- [ ] 首页可进入记录列表、新增记录、报告页
- [ ] 本地数据库可正常读写，不出现启动即崩溃
- [ ] 关键页面无明显白屏、空白容器或死循环加载
- [ ] 日志中无持续刷新的严重错误

## 核心闭环验证

- [ ] 新增一条仅文本记录，成功返回列表并看到“记录已保存”
- [ ] 打开刚创建的记录详情，能看到开始时间、结束时间、原始文本、摘要和备注 / 暂无备注
- [ ] 若本次演示包含图片附件，详情页能看到图片预览或明确的加载失败降级
- [ ] 生成一份周报、月报或季报，并能在列表中看到该报告
- [ ] 打开报告详情，能看到标题、摘要、建议和 Markdown 正文
- [ ] 关闭并重新打开应用后，已有记录和报告仍可读取

## 监控项

当前没有正式监控平台。发布 / 演示阶段只能做以下人工观察：

- 关键命令是否通过
- 应用日志是否出现连续网络或数据库异常
- UI 是否出现主链路级别的失败提示

## 回滚条件

- 首页或主入口无法打开
- 新增记录主链路连续失败
- 报告生成主链路连续失败
- 数据升级后旧记录不可读
- 本地数据库或附件读取导致应用崩溃

当前没有自动回滚系统；若触发以上条件，只能回退到上一版可运行构建或暂停当前演示版发出。
