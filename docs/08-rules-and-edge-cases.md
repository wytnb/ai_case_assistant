# 规则与边界

## 规则清单

| 规则编号 | 规则描述 | 优先级 | 备注 |
|---|---|---|---|
| R-01 | 新增记录时 `rawText.trim()` 必填 | 高 | 表单层先校验 |
| R-02 | 新增记录时 `rawText.trim().length` 不能超过 1000 | 高 | 前端阻止提交，服务层防御校验 |
| R-03 | `/ai/extract` 请求体必须同时发送 `rawText` 与 `eventTime` | 高 | `eventTime` 来自本次创建时捕获的客户端本地时间 |
| R-04 | `/ai/extract` 的 `eventTime` 必须是带 `+08:00`、且不带毫秒的 ISO 8601 字符串 | 高 | 例如 `2026-03-18T10:20:30+08:00` |
| R-05 | `notes` 缺失时按 `null` 保存 | 高 | 不补客户端伪造文案 |
| R-06 | `symptomSummary` 缺失时客户端回退为 `rawText` 首句 | 中 | 最多 36 字符加省略号 |
| R-07 | `sourceType` 当前统一保存为 `text` | 中 | 即使附带图片也不写 `mixed` |
| R-08 | 图片附件必须复制到应用私有目录后再入库 | 高 | 不依赖相册临时路径 |
| R-09 | 报告范围按最近 7 / 30 / 90 天计算 | 高 | 结束时间固定到当天结束 |
| R-10 | 参与报告筛选时只看 `createdAt` | 高 | `createdAt` 同时承担业务 `eventTime` 语义 |
| R-11 | 相同范围的报告按覆盖更新处理 | 高 | 删除多余重复记录 |
| R-12 | `adviceJson` 解析失败时页面降级为空建议列表 | 中 | 详情页不能崩溃 |

## 优先级

- 与保存正确性、契约一致性和数据一致性相关的规则优先级最高
- 与展示层降级相关的规则次之
- 与未来扩展性相关但不影响当前闭环的规则优先级最低

## 边界条件

- `rawText` 只有空白字符
  - 预期行为：表单提示“请输入原始描述”，不发起请求
- `rawText.trim().length == 1000`
  - 预期行为：允许提交
- `rawText.trim().length == 1001`
  - 预期行为：表单提示“原始描述不能超过1000字”，且服务层也拒绝请求
- `/ai/extract` 未返回 `notes`
  - 预期行为：记录仍可保存，详情页显示“暂无备注”
- `/ai/extract` 返回的 `symptomSummary` 为空或不可用
  - 预期行为：回退到 `rawText` 首句作为摘要
- 用户选择了图片，但复制文件失败
  - 预期行为：创建流程失败，已复制文件回滚删除
- 报告详情中的 `adviceJson` 不是合法 JSON 数组
  - 预期行为：建议列表为空，页面其他内容正常显示
- 详情页附件文件已不存在
  - 预期行为：图片区显示“图片加载失败”的降级文案

## 冲突处理

- 当 “AI 提取成功” 与 “客户端输入非法” 冲突时，以客户端输入校验为准，不发送请求
- 当 “AI 提取成功” 与 “AI 返回 payload 非法” 冲突时，以 payload 合法性为准，整条创建流程失败
- 当 “列表有数据” 与 “详情对象缺失” 冲突时，以详情查询结果为准，详情页显示未找到说明
- 当 “同范围已有报告” 与 “重新生成报告” 冲突时，以最新生成结果为准，旧结果被覆盖

## 异常场景

- 网络异常导致 `/ai/extract` 或 `/ai/report` 失败
  - 系统行为：页面提示失败，不崩溃
- 数据库读写失败
  - 系统行为：错误向上抛出，由页面显示失败提示
- 报告生成时 `events` 为空
  - 系统行为：客户端仍会请求 `/ai/report`
  - 当前状态：上游应返回空报告还是失败，仓库内未固定，属于待确认边界
- 真实 AI 集成测试未显式开启
  - 系统行为：相关测试默认跳过，不能写成“默认已覆盖”

## 默认兜底行为

- 无法从 `/ai/extract` 读取有效摘要时，回退到 `rawText` 首句
- 无法解析 `notes` 时，按空值处理
- 无法加载记录或报告列表时，显示错误态和重试入口
- 无法解析报告建议时，显示空建议而非抛异常
- 无法确认的上游行为统一写为“待确认”，不伪造为已确认规则
