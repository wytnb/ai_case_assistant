# 项目概览

## 背景

AI 健康病例助手当前是一个 monorepo：

- `apps/ai_case_assistant/` 承载 Flutter 客户端
- `services/ai_gateway/` 承载 AI gateway
- 根目录承载 workspace 文档、脚本与共享契约

当前目标不是做完整医疗平台，而是交付一个可真实演示、可本地落库、可接入 AI 整理与报告生成的 MVP。

## 目标用户

- 重视个人健康管理的普通用户
- 需要持续记录症状、检查结果、就诊线索和图片材料的人
- 希望在就医前快速整理近期健康信息的人

当前仓库不是为医院、医生工作站、保险理赔或科研统计场景设计。

## 目标问题

- 让用户能够低门槛记录健康相关原始输入
- 让系统把原始输入整理成可回顾的结构化健康事件
- 让用户能够按时间范围生成周期性总结，而不是只保留孤立记录
- 让 app 与 gateway 围绕同一份共享契约持续协作，避免接口漂移

## 核心价值

- 对用户的价值：减少健康信息散落与回忆成本，提升回顾效率
- 对项目的价值：提供一个本地优先、AI 协作、monorepo 管理的 Flutter MVP 示例仓库

## 成功标准

1. 新增记录主链路可演示
2. 本地数据可落库、可回显、可再次打开查看
3. AI 成功与失败两种情况对用户都可感知
4. 报告可按最近 7 / 30 / 90 天生成并查看
5. app、gateway、文档、测试围绕同一契约保持一致

## 当前版本目标

当前版本目标是一个 Android 优先的本地优先 MVP，覆盖：

- 首页入口
- 健康记录新增、列表、详情
- 图片附件本地保存与详情回显
- `POST /ai/intake` 驱动的追问 / 收口主链路
- `POST /ai/report` 驱动的周报、月报、季报生成与查看
- monorepo 根级文档、共享契约与轻量一致性校验

当前版本明确不以以下能力为目标：

- 账号体系
- 云同步
- OCR 或图片内容识别
- 通用化 AI 平台
- 记录编辑
- 权限系统
